## Overview

A project groups related issues toward a larger outcome — a launch, a rework,
a milestone-driven initiative. Where cycles slice work by time, projects slice
it by goal, and a project can span multiple teams.

![](/img/docs/projects.png)

## Create a project

Press the **+** next to **Projects** in the sidebar (or use the command bar).
A project has:

- **Name and description** — the description is a full rich-text document, so
  you can keep the plan, context, and decisions with the project itself.
- **Status** — e.g. Backlog, Planned, In Progress, Completed.
- **Dates** — start and target dates for tracking against a deadline.
- **Teams** — the teams whose issues can belong to this project.

## Working with projects

- **Assign issues:** Set the project from an issue's properties panel, or when
  creating the issue.
- **Milestones:** Break a project into milestones and assign issues to them to
  track intermediate targets.
- **Progress:** The project view shows completion progress based on the status
  of its issues, alongside the filtered list of everything in the project.
