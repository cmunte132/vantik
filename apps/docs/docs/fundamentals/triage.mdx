## Overview

Triage is your central hub for managing incoming requests. It captures issues
from sources beyond your team — GitHub issues, emails, monitoring tools, or
agents filing work through the API — so nothing lands directly in a team's
backlog without review.

## Perfect for

- **External requests:** Feature requests and bug reports arriving by email or
  through integrations land in one place instead of scattering across inboxes.
- **Open source management:** Manage incoming GitHub issues alongside other
  requests.
- **Agent-filed work:** Issues created by agents through the API can be routed
  through triage, giving a human a review point before work is accepted.

## Key benefits

- **Consolidated inbox:** All incoming requests flow into a single, organised
  location per team.
- **Review before commit:** Nothing enters the team's workflow until someone
  accepts it, keeping the backlog intentional.

## Review & manage triage issues

- **Scan & assess:** Glance through the queued issues one by one.
- **Action decisions:**
  - **Decline** requests that are irrelevant or invalid.
  - **Accept** valid requests and set their status (e.g. "Todo",
    "In Progress"), along with metadata like assignee or priority.
