## Overview

Vantik integrates with GitHub to keep your development process in sync:

- **Effortless linking:** Connect your pull requests (PRs) to related Vantik
  issues. Once a PR is closed, the linked Vantik issue automatically moves to
  "Done".
- **Automated issue creation:** New GitHub issues automatically create
  corresponding Vantik issues in triage.
- **Bidirectional sync:** Changes made in either platform (issue closure,
  comments, and other updates) automatically reflect in the other.

:::note
The GitHub integration ships as an [Action](/docs/actions/overview), so it
requires trigger.dev to be configured — see
[background jobs](/docs/oss/self-deployment#background-jobs-triggerdev).
:::

## Configure

### Connect GitHub

1. Navigate to **Settings → Overview → Integrations → GitHub**.
2. Grant Vantik access to your GitHub account.
3. Select the GitHub organisation you want to connect, and choose all
   repositories or specific ones to sync.

### Configure the action

1. Go to **Settings → Overview → Actions → GitHub → Configuration**.
2. Specify the repository, the Vantik team, and the label that triggers
   creating a GitHub issue from a Vantik issue.

Notes:

- Only one GitHub repository can be linked to each Vantik team.
- A single trigger label is used across all linked repositories and teams.
