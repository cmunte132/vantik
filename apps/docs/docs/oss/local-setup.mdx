## Overview

For development you run only the backing services (PostgreSQL, Redis,
SuperTokens, Typesense) in containers and the server + webapp on your host
with hot reload. The backing services publish their ports on localhost for
exactly this.

Prerequisites:

- Node.js ≥ 20
- pnpm 10 (`npm i -g pnpm@10`)
- Docker, or Podman with the compose provider

## Steps

1. Clone the repository and create your `.env` (the defaults work for local
   development):

```bash
git clone https://github.com/cmunte132/vantik.git
cd vantik
cp .env.example .env
```

2. Start the backing services only (this skips the webapp/server containers):

```bash
docker compose up -d postgres redis supertokens typesense
```

3. Install dependencies and apply the database schema:

```bash
pnpm install
pnpm migrate
```

4. Run the server (port 3001) and webapp (port 3000) with hot reload:

```bash
pnpm dev
```

:::tip
If the full stack is already running in containers, free the app ports first
with `docker compose stop webapp server`.
:::

5. **Access the application:** Navigate to
   [http://localhost:3000](http://localhost:3000). You'll be greeted with a
   login screen — enter your email address.

![](/img/docs/enter-email.png)

6. **Retrieve the magic link:** Without SMTP credentials in your `.env`, the
   magic link isn't emailed — it's printed in the server log from step 4:

```text
Sending email to dev@example.com with subject Log in to Vantik
Magic link: http://localhost:3000/auth/verify?token=...
```

   Copy the link into your browser to sign in.

7. Provide your details to set up your account and workspace:

![](/img/docs/account-setup.png)

8. **Create your first issue:** Your workspace should now look like this —
   press `C` to create your first issue.

![](/img/docs/default-workspace.png)
